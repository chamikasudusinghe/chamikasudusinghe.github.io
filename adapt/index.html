Accelerating Data Processing

The overarching goal of this strand of research is to accelerate data-processing tasks, with a focus on dataframes and the Pandas API. To this end, we develop technology to optimize and benchmark dataframe-backed data-processing systems.
Dias
Dias is an automatic rewriter for pandas code. It rewrites pandas code to semantically equivalent but faster versions, on the fly, transparently, and correctly. Dias is extremely lightweight and it will incur virtually no extra runtime or memory overheads. At the same time, Dias can provide 100× or even 1000× speedups.
SIGMOD 2024
PandasBench
PandasBench is the first systematic effort to create a benchmark for the Pandas API, the largest Pandas API benchmark to date, with 102 notebooks and 3,721 cells. We used it to evaluate Modin, Dask, Koalas, and Dias, creating the largest-scale evaluation of any of these techniques to date. We show that slowdowns are the norm, with Modin speeding up 8/102 (~8%) notebooks, and 0 for both Koalas and Dask. We also identified many failures of these systems: Modin runs only 72/102 (~70%) notebooks, Dask 4 (~4%), Koalas 10 (~10%), and Dias 97 (~95%).

TensorRight
TensorRight employs this algorithm to generate a finite number of bounded-verification proof obligations, which are then dispatched to an SMT solver using symbolic execution to automatically verify the correctness of the rewrite rules. We evaluate TensorRight’s verification capabilities by implementing rewrite rules present in XLA’s algebraic simplifier. The results demonstrate that TensorRight can prove the correctness of 115 out of 175 rules in their full generality, while the closest automatic, bounded-verification system can express only 18 of these rules.
PandasBench
In this paper, we introduce requirements that are necessary for a Pandas API benchmark, and present the first benchmark that fulfills them: PandasBench. We argue that it should evaluate the real-world coverage of a technique. Yet, real-world coverage is not sufficient for a useful benchmark, and so we also: cleaned it from irrelevant code, adapted it for benchmark usage, and introduced input scaling. We claim that uniform scaling used in other benchmarks (e.g., TPC-H) is too coarse-grained for PandasBench, and use a non-uniform scaling scheme. PandasBench is the largest Pandas API benchmark to date, with 102 notebooks and 3,721 cells
